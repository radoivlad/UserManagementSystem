Using a person's id, additional information can be retrieved (adding specific endpoint):
  • /id - receive data of a specific person, identified by id
  • /id/job - receive data of a specific person's job details
  • /id/salary - receive data of a specific person's salary
  • /id/workexperience - receive data of a specific person's work experience
Access Swagger interface (or use Postman) to perform additional operations in 'person' database, such as:
POST:
  • add a new person, insert: id, name, email, jobId and salaryIndex
PUT:
  • /id/salaryIndex - insert a person id, and a new salary index, to have it updated
DELETE:
  • /id - delete a specific person from the database, inputting id
